Event Name
Flow
URI Format
Raised on Object
RecordAnswer Survey /TenantName/Survey/RecordAnswer Survey Object for which answer is recorded
Summary
This is a demo event that shows how the same event can be customized to act differently for different tenants. In this demo, the RecordAnswer expects a email for SurveyDemo2 but not for SurveyDemo1 tenant.
Request Format
{
  "answers": [
    List of answers to be sent, in the order in which the questionaire was created.
    {
      "answer": "aye" or "nay" or "maybe"
    }
  ],
  "email" : "email of the user"
  "Survey": {
    "___smart_action___": "lookup",
    "___smart_value___": "Questionaire key"
  }
}
Request Description
Name
Value
Description
answers a list of answers The answers are recorded in the same order in which the questionaire was created.
email Email of user If a question indicates that email is required, then email has to be provided. Only if the advanced feature is enabled will this be enforced.
Response Format
{
  "event": "Event id for the request",
  "responses": [
    {
      "done": "Done",
      "___smart_responseid___": "Response id"
    }
  ]
}
Response Description
Name
Value
Description
done Done message Indicates that this answer has been recorded
