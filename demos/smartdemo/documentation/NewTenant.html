NewTenant AdminSmartFlow /SmartOwner/AdminSmartFlow/NewTenant TenantAdmin "SmartOwner"
Summary
SmartOwner is the default tenant created in every SMART platform and is used to administer SMART. The NewTenant event is used to create a new tenant in SMART. This sets up an isolated runtime environment for the tenant. By default the tenant comes with the standard flows - Security, AllFlows, AdminSmartFlow and Monitor. When a tenant is created, a custom flow can be enabled for the tenant. If not, the EnableFlow event can be called to enable a flow. The first flow enabled cannot be mashed up with anything and has to be an independent flow.
Request Format
{
  "TenantAdmin": {
    "___smart_action___": "lookup",
    "___smart_value___": "SmartOwner"
  },
  "tenant": "Unique name for tenant",
  "domain": "The domain name",
  "enableFlow": "Name of Flow to enable",
  "clientOf": "The third party name"
  "enableFeatures": [
    "feature of the flow"
  ],
}
Request Description
Name
Value
Description
tenant Tenant Name The unique name for this tenant. This is used in all the URIs for posting events.
domain domain name This is an optional field. This specifies the domain from which this tenant can be accessed. The Origin for the request is checked against this for validity.
clientOf name If the tenant is maintained by a third party, this is the third party name. A third party can create more than one tenant for use.
enableFlow flow name The name of a deployed flow that has to be by default enabled for this tenant.
features List of feature names A list of features in the flow that has to be enabled for this tenant by default.
