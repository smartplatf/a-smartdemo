Summary
SMART provides out of the box security that allows users to be associated to roles and access granted based on roles. SMART hence provides a user functionality that allows to create a single user and associate multiple identities to the same user. The user can login using any of the identities. The AddIdentity event creates an identity for the user. Unless an identity is added no permissions can be used.

{

  "credentialKey": "password",
  "type": "custom"
}
Request Description
Name
Value
Description
identity identity to add The identity with which this user can login. This can be linked to other authentication systems. Currently only custom supported.f
credentialKey password This is a password to be assigned for this identity.
type custom This indicates the type of identity. Currently only SMART validated users are supported. This will be extended in the future to support other open id authentications.
