Flow
URI Format
Raised on Object
EnableFlow AdminSmartFlow /SmartOwner/AdminSmartFlow/EnableFlow TenantAdmin "SmartOwner"
Summary
SmartOwner is the default tenant created in every SMART platform and is used to administer SMART. The EnableFlow is used to enable deployed flows for an existing tenant. This can be used to mashup one flow with another flow. During mashup it is assumed that the mashed flow is already enabled for this tenant. If not enabled, the system will give an error when executing the event.
Request Format
{
  "TenantAdmin": {
    "___smart_action___": "lookup",
    "___smart_value___": "SmartOwner"
  },
  "tenant": "Name of the tenant for which to enable",
  "enableFlow": "Flow to enable",
  "enableFeatures": [
    "Feature name to enable"
  ],
  "links": [
     provide list of links here.
    {
      "name": "name of the link",
      "flow": "flow to link to",
      "object": "object to link to",
      "attribute": "attribute to link to"
    }
  ]
}
Request Description
Name
Value
Description
tenant Tenant Name The tenant name for which to enable this flow.
enableFlow Name of the flow to enable Name of the flow that has to be enabled for this tenant.
enableFeatures List of feature names The list of feature names that has to be enabled for this tenant and is present in this flow.
links List of links If the flow enabled has needlinks then this is provided. Links can be to fill needlinks that related one object in this flow to an object in another flow, or a transition service that is called in this flow from another flow, or a data on which an event in this flow is raised. In the first case, the attribute is a member variable of the object. For the second, the attribute is a function name in a transition. In the third case attribute is not used.

{
  "event": "Event id for the request",
  "responses": [

      "___smart_responseid___": "response id"
    }
  ]
}

Name
Value
Description
