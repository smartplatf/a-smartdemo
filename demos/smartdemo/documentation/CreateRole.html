Flow
URI Format

CreateRole Security /TenantName/Security/CreateRole FlowAdmin "Security"
Summary
SMART provides out of box role definitions that can be created and assigned to users. This defines the access that a user can have to the different features in the flows. By default all objects created in SMART has a role based access defined unless overridden. This event is an out of box event provided by SMART to create roles. Roles are defined as a collection of access granted to different features in a flow.f
Request Format
{
  "FlowAdmin": {
    "___smart_action___": "lookup",
    "___smart_value___": "Security"
  },
  "roleName": "Role Name",
  "permits": {
    List of uri : permits.
  }
}
Request Description
Name
Value
Description
roleName name A name to be associated with the role being created. This has to be unique for the tenant.
permits map of uri and permits for uri This defines the permissions allowed in this role. A uri is a path indicating the flowname and the feature name and is of the format /flowname/featurename. The permit can be read, write or execute.
