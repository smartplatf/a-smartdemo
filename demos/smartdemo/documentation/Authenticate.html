Event Name
Flow
URI Format
Raised on Object
Authenticate Security /TenantName/Security/Authenticate FlowAdmin "Security"
Summary
The Authenticate Event is used to authenticate a user before accessing the role based features of any flow. Once authenticated, this event returns a session id which is expected in all subsequent requests. The role played by the user is then determined by the session id to grant access.
Request Format
{
  "FlowAdmin": {
    "___smart_action___": "lookup",
    "___smart_value___": "Security"
  },
  "identity": "User Name",
  "password": "Password",
  "type": "custom"
}
Request Description
Name
Value
Description
identity User name The user's identity that is used to authenticate the user
password password The password for the identity to authenticate. Currently this is not encrypted, when https is enabled, this will be encrypted.
type custom This indicates the type of authentication to do. Currently only custom is supported. This will change as and when facebook, gmail etc integration is done.
Response Format
{
  "event": "Event id of the request",
  "responses": [
    {
      "sessionId": "Session id allocated for this session",
      "___smart_responseid___": "Response id"
    }
  ]
}
Response Description
Name
Value
Description
sessionId session id The session id allocated for the authenticated session
